creating repos

three basic ways to create a repo

  • init
  • clone
  • fork

init

use init to create a repo

    
      $ git init

This command creates the necessary .git folder at the root of your project

    
      $ ls -la .git
      drwxr-xr-x  14 zackkitzmiller  staff  476 Jan 24 14:08 .

clone

use clone to pull down an existing repo you have read/read-write perms to

    
      $ git clone git@github.com:zackkitzmiller/Rockford-Dot-NET.git rockforddn

like init, this command creates the necessary .git folder at the root of your project

it also pulls down the current 'master' branch of the repo
